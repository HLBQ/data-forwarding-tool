# 数据转发器 (Data Forwarder)

## 简介

数据转发器是一个简单易用的网络代理工具，主要用于辅助手机等设备通过电脑进行网络代理。它可以将手机的网络请求转发到电脑上运行的代理服务，从而实现手机使用电脑代理的功能。

## 主要功能

- **数据转发**：将指定端口的网络流量转发到目标服务器
- **图形界面**：提供直观的GUI界面，方便配置和监控
- **实时统计**：显示连接数、数据传输量、网络速度等实时数据
- **日志记录**：详细的日志记录和查看功能

## 快速开始

## 使用方法

### 1. 基本配置

1. 启动程序后，在"转发配置"区域设置：
   - **监听端口**：电脑上监听的端口（默认：8070）
   - **目标主机**：要转发到的服务器地址（默认：127.0.0.1）
   - **目标端口**：目标服务器的端口号

2. 其他可选配置：
   - **最大连接数**：同时处理的最大连接数
   - **日志级别**：控制日志详细程度
   - **自动启动**：程序启动时自动运行转发器
   - **自动开放端口**：自动配置Windows防火墙规则
   - **清空日志**：每次启动时清空日志文件

### 2. 启动转发器

1. 点击"启动转发器"按钮
2. 程序会显示"状态: 运行中"
3. 日志区域会显示启动信息和连接状态

### 3. 手机配置

1. 确保手机和电脑在同一网络
2. 在手机的Wi-Fi设置或代理应用中配置：
   - **代理服务器**：电脑的IP地址
   - **代理端口**：设置的监听端口（默认8070）
   - **代理类型**：HTTP/SOCKS（根据目标代理服务类型）

### 4. 监控状态

- **实时统计**：查看活跃连接数、数据传输量、网络速度
- **IP统计**：查看连接设备的IP地址和访问记录
- **系统状态**：监控CPU、内存和网络使用情况
- **日志查看**：实时查看运行日志

### 5. 停止转发器

点击"停止转发器"按钮即可停止服务。

## 配置文件

程序使用 `forwarder_config.json` 文件保存配置，位置与可执行文件相同。配置文件包含以下选项：

```json
{
    "listen_port": 8070,
    "target_host": "127.0.0.1",
    "target_port": 53432,
    "max_connections": 100,
    "log_level": "ALL",
    "auto_start": false,
    "auto_open_ports": true,
    "clear_log_on_start": true
}
```

## 文件说明

- `forwarder_gui.py` - 图形界面主程序
- `data_forwarding_tool.py` - 核心转发功能
- `forwarder_config.json` - 配置文件
- `forwarder.log` - 运行日志
- `forwarder_ips.log` - IP地址日志

## 注意事项

1. **防火墙设置**：如果启用"自动开放端口"，程序会尝试配置Windows防火墙规则
2. **管理员权限**：自动配置防火墙可能需要管理员权限
3. **端口冲突**：确保监听端口没有被其他程序占用
4. **网络环境**：确保手机和电脑在同一局域网内
5. **目标服务**：确保目标代理服务正在运行

### 系统要求

- **最低配置**：
  - Windows 10 64位
  - 4GB RAM
  - 100MB 可用磁盘空间
  - 网络连接

- **推荐配置**：
  - Windows 10/11 64位
  - 8GB RAM
  - 1GB 可用磁盘空间
  - 稳定的网络连接



## 常见问题

### Q: 手机无法连接
- 检查电脑防火墙设置
- 确认手机和电脑在同一网络
- 检查监听端口是否正确
